# 快速开始

## Docker Compose

项目根目录已经提供 `docker-compose.yml`：

```bash
docker compose up -d --build
docker compose logs -f
```

启动后访问 [http://localhost:8889](http://localhost:8889)。停止服务使用：

```bash
docker compose down
```

普通停止不会删除数据库。只有执行 `docker compose down -v` 才会同时删除 `gentry-data` 数据卷。

## 本地运行

环境要求：

- Go 1.26
- Node.js 22 或兼容版本
- pnpm

构建前端并编译后端：

```bash
make build
make run
```

Windows 也可以运行根目录的 `build-windows.bat`。

默认地址为 [http://localhost:8080](http://localhost:8080)。可通过 `PORT` 环境变量修改端口。

## 开发模式

后端：

```bash
make dev
```

另一个终端启动前端：

```bash
cd frontend
pnpm install
pnpm run dev
```

开发界面默认运行在 [http://localhost:5173](http://localhost:5173)。

## 创建网页新增监控

1. 打开“新增监控”。
2. 选择“网页新增监控”。
3. 填写名称、网页 URL、分组和检查间隔。
4. 在“网站内容检测”中输入关键词并执行预扫描。
5. 从样本中选择正确候选；候选卡片上会标注策略来源（规则命中显示红色标签，启发式策略显示策略类型）。必要时展开"高级设置"调整容器、列表项和字段。
6. 选择推送全部新内容或仅推送命中关键词的内容，并选择通知账户。
7. 执行配置验证，确认能够稳定提取条目后保存。

首次检查只记录当前条目，之后出现新的稳定条目标识时才产生通知。

## 创建价格监控

1. 选择“价格监控”。
2. 选择单商品详情页或商品列表页。
3. 配置价格字段，并确保它能被解析为金额。
4. 单商品页可使用页面 URL 作为商品身份；列表页必须配置稳定且唯一的身份字段，例如 SKU 或商品链接。
5. 选择“价格发生下降”或“降到目标价及以下”。
6. 验证价格样本后保存。

首次检查只建立价格基线，不会因为页面当前已经是低价而立即推送。

## 保存扫描规则

当一个页面已经能够正确预扫描时，可以把结果保存为规则，供以后配置相同结构的页面：

1. 打开“扫描规则”。
2. 输入 URL 和关键词并执行预扫描。
3. 选择样本正确的候选区域。
4. 输入规则名称。
5. 选择“当前页面”“当前路由”或“通用结构”后保存。

默认选择“当前页面”。只有同一路由的子页面确实使用相同结构时才选择“当前路由”；只有多个网站使用同一套结构时才选择“通用结构”。

规则页面支持导出和导入 JSON。导入时同名规则会跳过，不会覆盖现有规则。

## 配置通知

1. 打开“推送管理”并启用全局通知。
2. 新建 PushPlus、Webhook、Server酱或 Bark 账户。
3. 在新增监控或监控详情中勾选需要使用的账户。

没有选择推送账户时，系统仍会保存变化记录，但不会发送通知。

## 日常操作

仪表盘可以启动、暂停、编辑或删除监控器。进入监控详情后，可以立即检查、重新建立基线、调整推送账户，并查看更新历史；价格监控还会显示当前快照和价格变动历史。
