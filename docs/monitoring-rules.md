# 监控规则说明

Gentry 将网页抓取结果转换为带稳定身份的观测记录，再与上一次快照比较。不同监控类型共用抓取、提取、规范化、事件持久化和通知投递链路。

## 网页新增监控

网页新增监控使用 `presence` 策略，适合公告、新闻、更新日志和商品上新列表。

- 首次成功检查建立基线，不推送页面中已有条目。
- 后续出现新的 `item_key` 时产生新增事件。
- 列表内容必须具有稳定身份，不能依赖随排序变化的位置索引。

常见身份字段包括文章 URL、公告编号和商品 SKU。

## 价格下降监控

价格监控使用 `field_transition` 策略，当前支持金额字段。

“价格发生下降”会在当前有效价格低于上一次有效价格时触发。可以配置：

- 任意降价即通知；
- 最低降价金额；
- 最低降价百分比；
- 同时满足最低金额和最低百分比。

价格短暂解析失败时，不会用无效值覆盖最后一次有效基线。币种发生变化时也不会直接跨币种比较。

## 到价提醒

“降到目标价及以下”采用跨边界语义。目标价格为 `X` 时：

- `100 → X`：通知；
- `100 → 小于 X`：通知；
- `X → 更低价格`：不重复通知；
- `低于 X → 高于 X`：不通知，但重新进入等待状态；
- 再次从高于 X 降到 X 或以下：再次通知；
- 首次基线已经低于 X：不通知。

这种语义可以避免监控周期内持续低价造成重复推送。

## 商品身份

系统必须能够在两次检查中识别同一个商品：

- 单商品详情页可以使用网页 URL 作为身份。
- 商品列表页必须使用稳定且唯一的字段，例如 SKU、商品详情链接或商品 ID。
- 价格字段不能作为身份，因为价格本身会发生变化。
- 不允许使用 `item_0` 一类列表位置作为身份。

## 基线

新建监控、手动重置基线，或者修改 URL、选择器、字段、商品身份、价格字段和价格规则后，需要重新建立基线。

名称、检查间隔和通知账户等不影响检测语义的修改，不应触发基线重建。

## 页面限制

默认抓取器适合服务端直接返回完整 HTML 的页面。如果价格只能在浏览器执行 JavaScript 后出现，或者页面依赖登录、验证码和复杂风控，普通 HTTP 抓取可能无法获取有效数据。
